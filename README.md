# Educational AI Assistant

Проект образовательного AI-ассистента на Python.

Пользователь выбирает роль преподавателя и формат ответа, после чего приложение формирует системную инструкцию и отправляет запрос выбранной языковой модели.

Сейчас проект поддерживает:

- локальные модели через Ollama;
- Mistral API;
- роли преподавателя математики и истории;
- краткий и полный форматы ответа;
- настройку моделей, ролей и промптов через YAML;
- валидацию конфигурации с помощью Pydantic.

## Текущий статус

Проект находится на стадии рабочего консольного прототипа.

Реализованы:

- загрузка конфигурации из `config.yml`;
- типизированные настройки приложения и LLM;
- переключение между моделями через `llm_key`;
- генерация системного промпта из роли и шаблона;
- вызов OpenAI-compatible Chat Completions API;
- получение ответов от Ollama и Mistral API;
- загрузка API-ключей из `.env`.


### Основные модули

- `config.py` — Pydantic-модели и загрузка настроек из YAML.
- `llm_client.py` — создание клиента для выбранной LLM.
- `assistant.py` — сборка системного промпта и отправка запроса модели.
- `main.py` — консольная точка входа.
- `config.yml` — модели, роли, шаблоны ответов и настройки приложения.

## Требования

- Python 3.12 или 3.13;
- `uv`;
- Ollama для локального запуска или API-ключ Mistral.

## Установка

Клонируйте репозиторий:

```bash
git clone https://github.com/MataNerdy/edu-assistant.git
cd edu-assistant
```

Установите зависимости:

```bash
uv sync
```

## Настройка Mistral API

Создайте файл `.env` в корне проекта:

```env
MISTRAL_API_KEY=your_mistral_api_key
```

Файл `.env` не должен попадать в Git.

Пример конфигурации модели:

```yaml
llms:
  mistral:
    model: "mistral-small-latest"
    base_url: "https://api.mistral.ai/v1"
    api_key_env: "MISTRAL_API_KEY"
    timeout: 60
    max_output_tokens: 512
```

Название модели можно изменить в `config.yml`.

## Настройка Ollama

Установите и запустите Ollama, затем загрузите модель:

```bash
ollama pull gemma3:1b
```

Пример локальной конфигурации:

```yaml
llms:
  ollama:
    model: "gemma3:1b"
    base_url: "http://127.0.0.1:11434/v1"
    api_key_env: null
    timeout: 60
    max_output_tokens: 512
```

Проверить, что Ollama работает:

```bash
curl http://127.0.0.1:11434/api/tags
```

## Роли ассистента

Сейчас определены две роли:

- `math_tutor` — преподаватель математики;
- `history_tutor` — преподаватель истории.

Инструкции ролей находятся в `config.yml`:

```yaml
roles:
  math_tutor:
    instruction: |
      Ты увлечённый учитель математики.
      Отвечай только на вопросы, связанные с математикой.

  history_tutor:
    instruction: |
      Ты — учитель истории.
      Давай структурированные ответы с датами.
      Отвечай только на вопросы, связанные с историей.
```

## Форматы ответа

Проект поддерживает два шаблона.

### `tutor_quick_answer`

Краткий ответ в одном предложении.

### `tutor_full_answer`

Развёрнутый ответ с объяснением, примерами и упражнениями.

Шаблоны задаются в `config.yml`, поэтому их можно менять без редактирования Python-кода.

## Запуск

В `main.py` выберите модель, роль и шаблон:

```python
from dotenv import load_dotenv

from edu_assistant.assistant import create_response

load_dotenv()

question = input()

response = create_response(
    llm_key="mistral",
    role="history_tutor",
    template="tutor_full_answer",
    prompt=question,
)

print(response)
```

Запустите приложение:

```bash
uv run main.py
```

После запуска введите вопрос в терминале.

Пример:

```text
Кто победил — половцы или печенеги?
```

Для использования локальной модели замените:

```python
llm_key="mistral"
```

на:

```python
llm_key="ollama"
```

## Конфигурация

Основные параметры моделей:

| Параметр | Назначение |
|---|---|
| `model` | Название модели |
| `base_url` | Адрес API |
| `api_key_env` | Название переменной окружения с API-ключом |
| `timeout` | Максимальное время ожидания ответа |
| `max_output_tokens` | Максимальная длина ответа |

## Ограничения текущей версии

Это учебный прототип. Модель может:

- допускать фактические ошибки;
- не всегда соблюдать ограничения роли;
- давать разные ответы на одинаковые вопросы;
- генерировать неточные упражнения или ответы.

Ответы ассистента желательно проверять, особенно при работе с историческими датами и сложными математическими задачами.
